# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Core Hibernation Engine.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope

This contract covers three UI surfaces delivered in Phase 1:

1. **Extension Popup** (`src/popup/App.tsx`) — 320px wide React popup rendered on icon click.
2. **Extension Icon Badge** — amber badge rendered by `chrome.action.setBadgeText()` in the Service Worker.
3. **Extension Icon** — static PNG assets at 16/32/48/128px.

Dashboard, thumbnails, placeholder pages, options page, and per-domain whitelist UI are explicitly out of scope for Phase 1 (deferred to Phase 2+).

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, tight inline padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, icon-to-label gap |
| md | 16px | `p-4` / `gap-4` | Default element spacing, popup section padding |
| lg | 24px | `p-6` / `gap-6` | Popup outer padding (top/bottom/left/right) |
| xl | 32px | `p-8` / `gap-8` | Reserved for Phase 2+ layouts |
| 2xl | 48px | `p-12` | Reserved for Phase 2+ layouts |
| 3xl | 64px | `p-16` | Reserved for Phase 2+ layouts |

**Popup outer padding:** `p-6` (24px all sides).

**Between popup sections (toggle row, button row, count row):** `gap-4` (16px vertical gap using flex-col).

**Exceptions:**
- Touch target minimum: 44px tall for all interactive controls (toggle row, button) — use `min-h-11` on the row wrapper. `min-h-11` is a static Tailwind class (44px = 11 × 4px), MV3 CSP compliant.
- Badge text: controlled by `chrome.action.setBadgeText()` — not a CSS concern.

## Color

| Role | Hex | Tailwind Class | Usage |
|------|-----|----------------|-------|
| Dominant (60%) | `#09090B` (zinc-950) | `bg-background` (shadcn CSS var) | Popup background — dark surface |
| Secondary (30%) | `#18181B` (zinc-900) | `bg-card` (shadcn CSS var) | Card/section containers within popup |
| Accent (10%) | `#F59E0B` (amber-400) | `text-amber-400` / `bg-amber-400` | Badge background, hibernated count number, active toggle thumb |
| Foreground | `#FAFAFA` (zinc-50) | `text-foreground` (shadcn CSS var) | All body text, labels |
| Muted foreground | `#A1A1AA` (zinc-400) | `text-muted-foreground` (shadcn CSS var) | Count label ("tabs hibernated"), disabled state text |
| Destructive | `#EF4444` (red-500) | `bg-destructive` (shadcn CSS var) | Not used in Phase 1 (no destructive actions) |

**Primary visual anchor:** The hibernated tab count number (`text-3xl font-semibold text-amber-400`) draws the eye first due to its large size and high-contrast amber color against the zinc-950 background. It is the dominant informational element in the popup body.

**Accent reserved for:**
1. Extension icon badge background (`#F59E0B` via `chrome.action.setBadgeBackgroundColor`) — already specified in RESEARCH.md Pattern 8.
2. Hibernated tab count number displayed in popup body.
3. Active state of the global on/off toggle thumb/track.
4. Active state of the "Protect this tab" toggle thumb/track.

**Accent is NOT used for:** button backgrounds, hover states, borders, or text in any other context.

**Dark mode:** Single dark theme only for Phase 1. The popup uses a dark surface (`zinc-950` background) to match the "power user / The Great Suspender successor" aesthetic. No light/dark toggle in Phase 1. Light mode deferred to Phase 5 (Polishing & Launch).

**Badge color locked:** `#F59E0B` — matches Tailwind `amber-400`. Already confirmed in RESEARCH.md Pattern 8 (`chrome.action.setBadgeBackgroundColor({ color: '#F59E0B' })`). Source: RESEARCH.md.

## Component Specifications

### 1. Popup Shell

```
width: 320px (fixed — Chrome extension popup standard)
padding: 24px (p-6)
background: bg-background (#09090B)
layout: flex flex-col gap-4
```

### 2. Popup Header

```
Layout: flex items-center gap-2
Icon: 20x20px extension icon (img tag pointing to icons/icon48.png, scaled via CSS)
Brand name: "Smart Hibernator" — text-base font-semibold text-foreground
No subtitle in Phase 1.
```

### 3. Global Hibernation Toggle Row

```
Layout: flex items-center justify-between min-h-11
Label: "Hibernation" — text-sm font-normal text-foreground
Sublabel: "Auto-sleep inactive tabs" — text-xs font-normal text-muted-foreground (below label)
Control: shadcn Switch component
  - Checked state: track bg-amber-400, thumb white
  - Unchecked state: track bg-zinc-700, thumb white
  - Storage key written: hibernation_enabled (boolean) in chrome.storage.local
  - Disabled state: opacity-50 cursor-not-allowed
```

### 4. "Hibernate this tab" Button

```
Component: shadcn Button, variant="default", size="default"
Width: w-full
Text: "Hibernate this tab" — text-sm font-normal
Background: bg-zinc-800 hover:bg-zinc-700 (NOT amber — accent is reserved)
Text color: text-foreground
Border: border border-zinc-700
Height: min-h-11
Disabled state (when tab is already hibernated OR tab is protected):
  opacity-50 cursor-not-allowed
  Text changes to "Already hibernated" or "Tab is protected" respectively
Icon: Moon icon (Lucide) at 16x16px, mr-2, before the label text
```

### 5. "Protect this tab" Toggle Row

```
Layout: flex items-center justify-between min-h-11
Label: "Protect this tab" — text-sm font-normal text-foreground
Sublabel: "Exclude from auto-hibernation" — text-xs font-normal text-muted-foreground
Control: shadcn Switch component (same style as global toggle)
  - Checked state: track bg-amber-400, thumb white
  - Unchecked state: track bg-zinc-700, thumb white
  - Storage key written: protected_tabs (array of tab IDs) in chrome.storage.local
```

### 6. Hibernated Count Display

```
Layout: flex flex-col items-center justify-center py-2
Count number: text-3xl font-semibold text-amber-400
Count label: "tabs hibernated" — text-xs font-normal text-muted-foreground mt-1
Container background: bg-card rounded-lg p-4
Empty state (count = 0): count shows "0" in text-zinc-600 (not amber — amber is reserved for non-zero counts)
```

### 7. Extension Icon Badge

```
Text: String(count) when count > 0; "" (empty) when count = 0
Max display: "999+" for counts >= 1000 (4-char limit per chrome.action API)
Background color: #F59E0B (amber-400)
Text color: Chrome default (white) — not configurable via web API
```

### 8. Dividers Between Sections

```
Use shadcn Separator component (thin 1px line)
Color: bg-zinc-800
Between: header and toggle section; between toggle section and button; between button and count display
```

## Extension Icon Design Direction

| Size | Usage | Design |
|------|-------|--------|
| 16px | Tab strip favicon, toolbar (small) | Solid amber circle with white crescent/moon silhouette |
| 32px | Standard toolbar | Same design, slightly more detail |
| 48px | Extension management page | Full icon with visible detail |
| 128px | Chrome Web Store listing | High-resolution version of same icon |

**Design language:** Minimal flat icon. Single amber circle (`#F59E0B`) as background. White crescent moon or "zzz" symbol in center to communicate "sleep/hibernation." No gradients, no shadows. The amber background directly ties the icon to the badge color — the same amber signals "hibernation is active" across both surfaces.

**File locations:** `icons/icon16.png`, `icons/icon32.png`, `icons/icon48.png`, `icons/icon128.png` (relative to project root, as required by CRXJS — no `./` prefix).

**Implementation note for Phase 1:** Placeholder icons (simple amber square with moon) are acceptable for development. Final icon polish is deferred to Phase 5.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Popup header brand | "Smart Hibernator" | Exact product name — no tagline in Phase 1 |
| Global toggle label | "Hibernation" | Noun, not a verb. Consistent with Chrome extension conventions |
| Global toggle sublabel | "Auto-sleep inactive tabs" | Clarifies what hibernation means |
| Protect toggle label | "Protect this tab" | Imperative noun phrase |
| Protect toggle sublabel | "Exclude from auto-hibernation" | Clarifies the effect |
| Hibernate button | "Hibernate this tab" | Primary CTA — verb + object. Source: CONTEXT.md D-03 |
| Button when tab is discarded | "Already hibernated" | Descriptive, not an error |
| Button when tab is protected | "Tab is protected" | Descriptive, not an error |
| Button loading state | "Hibernating..." | Progressive indicator |
| Count label (non-zero) | "tabs hibernated" | Plural always — simpler than singular/plural logic for Phase 1 |
| Count label (zero) | "No tabs hibernated" | Replaces count number entirely when 0 |
| Empty badge | "" (empty string) | Badge cleared when count = 0. Source: RESEARCH.md Pattern 8 |
| Context menu item 1 | "Hibernate this tab" | Must match popup button label — consistency |
| Context menu item 2 | "Protect this tab from hibernation" | Matches RESEARCH.md Pattern 7 |

**Destructive actions in Phase 1:** None. The "Hibernate this tab" action discards a tab but does NOT close it — the tab remains in the strip and can be restored by clicking it. No confirmation dialog required. This is a reversible action.

**Error states in Phase 1:**
- If `chrome.tabs.discard()` returns `undefined` (tab is protected/NTP/chrome://): button silently shows "Already hibernated" state without an error toast. No error UI needed — the tab was simply not discardable.
- If `chrome.storage.local` read fails on popup mount: show all toggles in their default (off) state. No error banner in Phase 1.
