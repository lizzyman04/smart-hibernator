<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Offscreen Document -> External network | NFR-04 invariant: NEVER call fetch() with non-chrome-extension URLs |
| Service Worker -> Offscreen Document | chrome.runtime.sendMessage; both endpoints same extension origin; no cross-extension messaging permitted |
| Tab content -> Service Worker | Existing Phase 1 boundary; SW only reads tab.url/title/favIconUrl provided by Chrome — never executes tab content scripts |
| chrome.storage.local ai_classifications | SW writes; popup + dashboard read; bounded by tab count (Pitfall 4) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-05 | Information Disclosure | Offscreen Document fetch() in classify path | HIGH | mitigate | Source-grep gate: classifier.ts and offscreen/main.ts must NOT call fetch() with anything other than chrome.runtime.getURL(...) results; test asserts via vi.spyOn(global, 'fetch') that all fetch URLs start with `chrome-extension://` |
| T-03-06 | Denial of Service | ai_classifications storage quota (Pitfall 4) | MEDIUM | mitigate | classifyBatch prunes stale entries (tabIds no longer in chrome.tabs.query result) before write; cap structure to `{ [tabId]: { label, confidence, cachedAt } }` (no large payloads) |
| T-03-07 | Tampering | Domain bias values read by classifier feature vector | LOW | mitigate | buildFeaturesForTab clamps `domainBiasOffset` to `[-1, 1]` before inclusion in feature vector (Math.max/Math.min); rejects NaN |
| T-03-08 | Elevation of Privilege | Untrusted message types on chrome.runtime.onMessage in offscreen | LOW | mitigate | Offscreen handler checks `message.type === 'CLASSIFY_BATCH'` strictly; ignores all other types; validates `Array.isArray(message.tabs)` and per-tab `features.length === 6` before running session.run; sender check would be redundant since chrome.runtime messages are origin-scoped to this extension |
| T-03-09 | Race Condition | Concurrent ensureOffscreen calls (Pitfall 2) | MEDIUM | mitigate | Module-level `creatingOffscreen` promise guard; second caller awaits first caller's promise; chrome.runtime.getContexts() check precedes createDocument |
| T-03-10 | Tampering | IndexedDB version upgrade race (Pitfall 3) | MEDIUM | mitigate | openDB v2 declares `blocked()` log and `blocking()` callback that nullifies module-level dbPromise so the other context can close its connection |
</threat_model>
