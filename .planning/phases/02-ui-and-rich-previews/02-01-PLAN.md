## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Elevation of Privilege | manifest.json `activeTab` permission | accept | activeTab is a temporary grant activated only by user gesture (tab load complete event). It is the minimum permission needed for captureVisibleTab per RESEARCH.md Pitfall 1. Using `<all_urls>` instead would trigger a scary CWS permission warning — activeTab avoids this. No mitigation needed beyond using activeTab. |
| T-02-01-02 | Elevation of Privilege | manifest.json `web_accessible_resources` `matches: ["<all_urls>"]` | accept | This is required for chrome.runtime.getURL to work from any page. The resource exposed is only the dashboard HTML file — no sensitive data. Per MV3 documentation this is the standard pattern for extension pages opened programmatically. |
| T-02-01-03 | Information Disclosure | manifest.json `style-src 'unsafe-inline'` in extension_pages CSP | accept | Scope-limited to extension pages ONLY (not content scripts). The relaxation enables Recharts Tooltip inline styling. script-src 'unsafe-eval' remains forbidden (MV3 minimum policy). Risk is low: extension pages are same-origin and not reachable by external web content. |
| T-02-01-04 | Tampering | fake-indexeddb in vitest.setup.ts | accept | This is a dev-only dependency that replaces the global indexedDB in the jsdom test environment. It does not run in production. No mitigation needed. |
</threat_model>
