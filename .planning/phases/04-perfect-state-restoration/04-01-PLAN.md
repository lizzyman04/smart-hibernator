## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| content script → SW (future) | Untrusted page DOM values flow toward IDB; this plan only defines the storage contract |
| page DOM → IDB record | Captured field values become persisted-to-disk data; type defines the shape that exclusion logic (04-03) must respect |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | TabStateSnapshot.fields[] persisted to IDB | mitigate | Type carries no field that would force capture of sensitive inputs; D-03 exclusion is enforced at capture time in 04-03. FieldSnapshot.value is plain string — no schema affordance for password/hidden persistence. |
| T-04-02 | Denial of Service | tab-state IDB store growth | mitigate | MAX_FIELDS (50) + MAX_FIELD_VALUE_LEN (10_000) constants defined here bound per-snapshot size; delete-after-restore + onRemoved eviction (04-02) bound store cardinality. No size-cap prune needed for this store. |
| T-04-03 | Tampering | IDB upgrade path | accept | v3 upgrade is additive (oldVersion < 3 branch only); existing stores untouched. blocked/blocking + CR-03 .close() preserved verbatim — no new tampering surface. |
</threat_model>
