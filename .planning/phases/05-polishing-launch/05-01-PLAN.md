<objective>
Implement the offscreen-document idle-teardown lifecycle (D-01) with the in-flight race guard (D-05) and a guarded dev memory probe (D-02 visibility half). This is the primary NFR-01 lever: the offscreen document holds the ONNX InferenceSession + ORT-Web WASM runtime + model buffer, the dominant steady-state RAM consumer. Closing the document destroys its JS+WASM context so the OS reclaims the heap.

Purpose: Drive total extension RAM toward the ≤45MB NFR-01 budget by releasing the biggest consumer when the browser is genuinely idle, while never breaking classification correctness (no teardown mid-inference; transparent recreate on next call).
Output: Extended `classifier.ts` lifecycle chokepoint, a `RELEASE_SESSION` handler in `offscreen/main.ts`, a new `OFFSCREEN_IDLE_MS` constant, a DEV-gated `mem-probe.ts`, and extended unit tests.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Denial of Service | classifier.ts idle teardown vs in-flight CLASSIFY_BATCH (the D-05 race) | mitigate | `pending` ref-count guard: teardownIfIdle early-returns when pending > 0; pending++ before send, pending-- in finally. Asserted by unit test. Worst case if wrong = a single lost/hung classification, recovered next alarm tick. |
| T-05-02 | Denial of Service | Idle window too short → cold-start thrash (Pitfall 1) violates NFR-02 ≤150ms warm | mitigate | OFFSCREEN_IDLE_MS = ~10 min, far above the 1-min alarm period, so the doc stays warm during active browsing and only tears down when genuinely idle. |
| T-05-03 | Information Disclosure | mem-probe leaking timing/memory data or shipping in prod | accept/mitigate | Probe is DEV-gated (import.meta.env.DEV) so production carries zero probe code; logs only to console.debug locally; no network egress (preserves NFR-04). |
| T-05-04 | Tampering | RELEASE_SESSION leaving a half-disposed session singleton | mitigate | finally block nulls BOTH session and sessionInit even if release() rejects, so a recreated doc re-inits cleanly. |
